Project Name: Budgetarian
Tech Stack: Java (backend), HTML, CSS (frontend), JSON (data storage or API exchange)
Type: Personal finance web app with warranty tracking (Warranty Keeper)

Core Description:
Build a fully functional website called Budgetarian that helps users take control of their finances. The application must include a login system so each user has their own private data. Use Java for server‑side logic (servlets, JSP, or Spring MVC), HTML/CSS for the frontend interface, and JSON for data storage like saving user data in .json files on the server, or exchanging data between frontend and backend via JSON APIs. All features below should be tied to the logged‑in user.

Authentication & Account Features:

Sign up / Log in – Users create an account with email and password. Passwords must be hashed (e.g., BCrypt) before storing in JSON files.

Session management – Use HTTP sessions (or JWT tokens stored in cookies/localStorage) to keep users logged in until logout.

Logout – Destroy session and redirect to login page.

Data storage – Each user's data (goals, expenses, installment entries, warranties, receipts metadata) is stored in a separate JSON file on the server, or in a single JSON file with user IDs as keys.

Optional profile – Show user email, allow account deletion (remove their JSON file).

Core Features (after login):

Savings Goals – Set weekly or monthly savings targets, track progress (stored in user's JSON).

Expense Tracking – Log and categorize daily expenses, view spending trends.

Money‑Saving Tips & Income Ideas – Display a curated list (can be stored in a separate tips.json file loaded by Java backend).

Paylater & Installment Tracker – Enter upcoming due dates. The system sends in‑app notifications (using Java backend to check due dates and display alerts on the dashboard) to avoid late fees.

Warranty Keeper – Log purchases, upload receipt images (store file path in JSON, actual image saved on server), set warranty/return expiration dates. Automatic reminders before expiry.

User Interface Requirements:

Responsive design – Works on desktop, tablet, and mobile using CSS media queries.

Login / Signup screens – Simple, card‑based HTML forms. On submission, Java backend validates credentials against JSON user store.

Dashboard (post‑login) – Shows savings progress, recent expenses, upcoming installment dues, active warranties (all fetched from user's JSON via Java servlets/APIs).

Navigation – Sidebar or top menu with icons: Home, Expense Tracking, Savings Goals, Tips & Ideas, Paylater Tracker, Warranty Keeper, Logout.

Forms & inputs – Use HTML5 date pickers, file upload for receipts. JavaScript (optional) for real‑time validation; backend validation in Java as well.

Notifications – Visual badges on dashboard (e.g., "3 upcoming dues") generated by Java backend reading JSON.

Accessibility – Sufficient color contrast, semantic HTML, keyboard navigable.

JSON File Structure (example):

users.json – Stores user credentials (hashed passwords, email, user ID).

user_{id}.json – Stores all user‑specific data (savings goals, expenses, installments, warranties, receipt paths).

tips.json – Static list of money‑saving tips and income ideas.

Java Backend Requirements:

Use Java Servlets (or Spring Boot if preferred) to handle HTTP requests (login, signup, add/update expense, add warranty, etc.).

All responses should be HTML pages (JSP or servlet‑generated) or JSON endpoints if using AJAX.

Use java.nio.file to read/write JSON files on the server.

For file uploads (receipts), use javax.servlet.http.Part or MultipartConfig.

Implement a simple in‑memory or file‑based session tracking.





Frontend (HTML/CSS):

No heavy frontend framework required – plain HTML, CSS, and optional vanilla JavaScript for better UX (e.g., dynamic form updates).

CSS file(s) implementing the Budgetarian color palette.

All pages share consistent header/footer.

Deliverable:
A complete, runnable web application (e.g., deployable on Tomcat or any Java servlet container) with all features listed above, using Java backend, HTML/CSS frontend, and JSON for persistent data storage. The login system must securely isolate user data.
